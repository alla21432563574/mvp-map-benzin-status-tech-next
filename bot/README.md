# Telegram-бот «Есть топливо»

Бот на Telegraf предлагает выбрать город и АЗС, умеет искать ближайшие станции по отправленной геолокации и сохраняет выбранные статусы топлива в `pending_reports`. Он никогда не меняет таблицу `stations` напрямую.

## Подготовка

1. Создайте бота через `@BotFather` и получите токен.
2. Примените `supabase/schema.sql` для новой базы или `supabase/migrations/20260703_add_telegram_bot.sql`, если базовая схема уже установлена.
3. В папке `bot` создайте конфигурацию:

```bash
cp .env.example .env
```

Заполните `TELEGRAM_BOT_TOKEN`, `SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY`. Бот работает как доверенный backend-процесс: публичная anon-вставка в `pending_reports` закрыта, чтобы отчёты нельзя было спамить напрямую через Supabase API.

## Запуск

```bash
cd bot
pnpm install
pnpm dev
```

Для обычного запуска без слежения за файлами используйте `pnpm start`.

Бот работает в long polling режиме, поэтому его нужно размещать как постоянно запущенный Node.js-процесс (например, Railway, Render worker, Fly.io или VPS). Vercel подходит для веб-приложения, но не для этого long polling процесса.

Команды бота:

- `/start` — приветствие и новый отчёт;
- `/report` — начать новый отчёт.
